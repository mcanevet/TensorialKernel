> ⚠️ **Disclaimer**: I absolutely don't know what I'm doing. I have no formal background in operating systems research. I am still very skeptical about whether an LLM could actually "invent" something meaningful in this field. The purpose of this project is NOT to publish anything seriously — it's purely a hobby project to satisfy my curiosity about how far I could push this concept. I plan to keep challenging it with newer, more performant models over time.

**I'd gladly accept contributions, prompts, suggestions, or any feedback** — whatever helps me see how far this can go. If you read this and think it's complete nonsense, please tell me — I'll learn something either way.

> A research project born from a simple question: *"What if an LLM could help me build an operating system from scratch?"*

## How It Started

It began with curiosity — the kind that makes you ask a machine: *"Can you help me create a simple Operating System from scratch?"*

After a few tens of minutes of prompts and iterations, I had a functional kernel running on QEMU. Victory!

But curiosity doesn't stop at "good enough." I wanted to go bigger. I asked: *"What's the state of the art in kernel architectures?"*

The LLM told me about monolithics, microkernels, and exokernels — the established paradigms that have governed operating system design for decades. Then I asked the question that changed everything:

> *"Could you invent a new kernel architecture?"*

And it did. It invented the concept of the **Tensorial Kernel** — a fundamentally different model where the kernel doesn't manage resources, but maintains **consistency invariants across heterogeneous compute elements**.

## The Skepticism

I was suspicious. Very suspicious. So I started challenging it — cross-challenging with other LLMs, improving the research, refining the prompts. What emerged was a formal contract: a rigorous definition of consistency primitives (`coherent`, `exclusive`, `relaxed`) grounded in established theory like fractional permissions and δ-consistency.

The research grew from a wild idea into a well-defined architecture with:

- **Consistency contracts** as the primary kernel API
- **Capability tensors** for multi-projection memory views
- **Witness chains** for runtime violation detection
- **Topological mapping** for data path optimization
